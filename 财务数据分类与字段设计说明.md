# 财务数据分类与字段设计说明

## 1. 设计目标

本分类体系用于个人/家庭财务数据的结构化管理，后续需要支持：

- 自动分类
- 自动统计分析
- 图表展示
- 月度/年度预算管理
- 收入、支出、转账、资产变动区分
- 按消费对象分析，如本人、子女、家庭、父母等

本体系适合用于 Excel、CSV、SQLite、PostgreSQL、Python、Power BI 等数据处理场景。

---

## 2. 总体结构

每一笔财务流水应采用以下核心结构：

```text
收支类型
├── 一级分类
│   ├── 二级分类
│   │   ├── 三级分类
```

同时增加以下分析字段：

```text
目标人
项目
标签
账户
支付方式
商户/对象
是否报销
是否预算内
原始摘要
```

其中：

- 分类字段用于树状归类
- 标签字段用于交叉分析
- 目标人字段用于区分钱花在谁身上
- 项目字段用于区分家庭、工作、装修、旅行等事项

## 3. 推荐数据字段

| 字段名 | 字段类型 | 是否必填 | 示例 | 说明 |
|---|---|---:|---|---|
| id | string/int | 是 | 202605180001 | 唯一流水编号 |
| 日期 | date | 是 | 2026-05-18 | 交易日期 |
| 年 | int | 是 | 2026 | 可由日期自动生成 |
| 月 | int | 是 | 5 | 可由日期自动生成 |
| 收支类型 | enum | 是 | 支出 | 收入、支出、转账、债权债务、资产变动 |
| 一级分类 | string | 是 | 居住生活 | 分类第一级 |
| 二级分类 | string | 是 | 水电能源 | 分类第二级 |
| 三级分类 | string | 否 | 电费 | 分类第三级 |
| 金额 | decimal | 是 | 356.20 | 交易金额，建议统一为正数 |
| 币种 | string | 否 | CNY | 人民币为 CNY，日元为 JPY |
| 账户 | string | 否 | 招商银行卡 | 资金账户 |
| 支付方式 | string | 否 | 微信支付 | 微信、支付宝、银行卡、现金、信用卡等 |
| 商户/对象 | string | 否 | 国家电网 | 收款方、付款方、商户或个人 |

| 项目 | string | 否 | 家庭日常 | 项目维度 |

| 原始摘要 | string | 否 | 微信支付-国家电网 | 银行或支付平台原始摘要 |
| 备注 | string | 否 | 4月电费 | 人工补充说明 |

## 4. 收支类型定义

| 收支类型 | 说明 | 是否进入日常收入/支出统计 |
|---|---|---:|
| 收入 | 工资、奖金、投资收益、副业收入等 | 是 |
| 支出 | 餐饮、居住、交通、医疗、教育等真实消费 | 是 |
| 转账 | 账户互转、信用卡还款、现金存取 | 否 |
| 债权债务 | 借出、借入、归还、收回 | 视分析口径决定 |
| 资产变动 | 基金买入、股票卖出、理财申购、赎回等 | 否 |

重要规则：

- 信用卡还款不作为普通支出
- 银行卡转支付宝不作为普通支出
- 基金买入不作为普通支出
- 贷款本金还款不作为普通支出
- 贷款利息可以作为支出
- 报销垫付应单独标记，分析真实消费时可排除

## 5. 目标人字段设计

### 5.1 字段名称

```text
目标人
```

程序字段名建议使用：

```text
target_person
```

### 5.2 字段含义

目标人表示该笔收入或支出主要对应的对象，即“这笔钱主要是为谁花的”或“这笔收入归属于谁”。

### 5.3 推荐枚举值

| 目标人 | 说明 |
|---|---|
| 本人 | 用户本人个人消费或收入 |
| 配偶 | 配偶相关消费或收入 |
| 子女 | 子女教育、医疗、服装、玩具等 |
| 父母 | 父母赡养、医疗、礼品等 |
| 家庭 | 全家共同使用或难以区分个人对象的支出 |
| 工作 | 因工作产生的支出或收入 |
| 朋友 | 请客、礼品、往来等 |
| 亲属 | 非父母、配偶、子女之外的亲属 |
| 其他 | 无法归类的对象 |

### 5.4 使用规则

#### 规则 1：能明确个人的，填具体个人对象

| 支出内容 | 目标人 |
|---|---|
| 自己午餐 | 本人 |
| 孩子学费 | 子女 |
| 给父母买药 | 父母 |
| 给配偶买礼物 | 配偶 |

#### 规则 2：全家共同使用的，填“家庭”

| 支出内容 | 目标人 |
|---|---|
| 家庭电费 | 家庭 |
| 家庭燃气费 | 家庭 |
| 家庭宽带费 | 家庭 |
| 全家一起吃饭 | 家庭 |
| 家庭日用品 | 家庭 |

#### 规则 3：工作垫付或工作用途，填“工作”

| 支出内容 | 目标人 |
|---|---|
| 出差打车 | 工作 |
| 替公司买办公用品 | 工作 |
| 商务宴请 | 工作 |
| 待报销酒店费 | 工作 |

#### 规则 4：多人共同受益但无法拆分时，填“家庭”或主要对象

| 支出内容 | 建议目标人 |
|---|---|
| 全家旅游 | 家庭 |
| 亲子游主要为孩子 | 子女 |
| 家庭聚餐 | 家庭 |
| 给孩子买学习桌，全家也使用 | 子女 |

#### 规则 5：一笔费用涉及多个目标人时，优先拆分

如果一笔交易中同时包含本人、子女、家庭等不同对象，建议拆分为多条记录。

原始交易：

```text
超市消费 500 元
其中：
- 家庭食品 300 元
- 子女零食 100 元
- 本人用品 100 元
```

应拆分为：

| 金额 | 一级分类 | 二级分类 | 三级分类 | 目标人 |
|---:|---|---|---|---|
| 300 | 餐饮食品 | 食材采购 | 蔬菜水果/肉蛋奶 | 家庭 |
| 100 | 餐饮食品 | 饮品零食 | 零食饮料 | 子女 |
| 100 | 购物消费 | 日用百货 | 个人用品 | 本人 |

如果无法拆分，则使用主要对象或“家庭”。

## 10. 自动分类处理规则

### 10.1 基础判断顺序

自动分类时建议按以下顺序判断：

```text
1. 判断是否为转账、信用卡还款、投资买卖
2. 判断是否为收入
3. 判断是否为报销或垫付
4. 判断是否为真实支出
5. 识别一级分类
6. 识别二级分类
7. 识别三级分类
8. 识别目标人
9. 识别标签
10. 输出结构化结果
```

### 10.2 分类优先级规则

#### 信用卡还款

如果原始摘要中包含：

```text
信用卡还款、还信用卡、卡还款
```

则：

```json
{
  "收支类型": "转账",
  "一级分类": "信用卡",
  "二级分类": "信用卡还款",
  "三级分类": "储蓄卡还信用卡"
}
```

不得计入普通支出。

#### 投资买入

如果原始摘要中包含：

```text
基金申购、基金买入、股票买入、理财申购
```

则：

```json
{
  "收支类型": "资产变动",
  "一级分类": "投资交易",
  "二级分类": "投资买入"
}
```

不得计入普通支出。

#### 报销到账

如果原始摘要中包含：

```text
报销、差旅报销、费用报销
```

且金额流入，则：

```json
{
  "收支类型": "收入",
  "一级分类": "工资薪酬",
  "二级分类": "报销到账",
  "是否报销": "已报销"
}
```

分析真实收入时可选择排除。

#### 工作垫付

如果消费内容属于工作用途，但尚未报销，则：

```json
{
  "收支类型": "支出",
  "一级分类": "工作商务",
  "二级分类": "报销垫付",
  "是否报销": "待报销",
  "目标人": "工作"
}
```

## 17. 核心原则

1. 分类不要过细，否则后期维护困难。
2. 年度预算建议做到二级分类。
3. 三级分类用于明细分析，不建议全面做预算。
4. 目标人用于回答“钱花在谁身上”。
5. 项目用于回答“钱花在哪件事上”。
6. 标签用于回答“这笔钱有什么属性”。
7. 转账、信用卡还款、投资买入不得计入普通支出。
8. 报销垫付应单独标记，分析真实家庭支出时可以排除。
9. 同一笔交易涉及多个目标人时，优先拆分。
10. 无法判断时，保留原始摘要并输出较低置信度。
